# 野性起源

- 术语
  - 灵能 魔法能源
  - 灵力 具体数值的魔法能量
  - 灵根、灵根耐力
  - 法术、功法
  - 天然灵石、数字灵石 货币
- 灵能、经验、附魔的 Cap
- menu screen
  - 多媒体容器菜单 api
  - 标签、页面、滚动条
  - 多样式，数据驱动，不同的机器可有不同的样式，但核心逻辑不变。
  - 无魔简时使用设备的样式，有魔简时使用魔简的样式。
  - 线式物品堆数值显示，减少遮挡。类似耐久条，条后方用点格代表单位
  - 合成格右边加一个格，可以放物品容器，会替代玩家库存 3 行，和上库存互动，拿下就恢复
- screen
  - 无样式，透明背景，极简，多用于快速选择等菜单比如手持装备切换配置
  - 快捷功能
- 辅助提示：
  - 本模组相关大部分提示，比如tooltip，但亦有基于服务端的消息提示，因此需要做同步
  - 适配原版高级提示框
- 快捷栏选择器
  - 最大 9x9（含原版）的快捷栏阵列，行为整个快捷栏
  - 需要附属解锁与控制，无附属时初始解锁
- 替代 TAB 菜单
  - 快捷菜单，可自定义各种元素，便捷按钮
- 团队功能
  - 联动 FtpTeam
  - 功能域：创建区域独立的团队规则，比如 PVP，磁铁开关。
  - 支持加入多个队伍，敌对属性优先
- 基于neo的新物流API的实用api
- 秘笈：多功能终端
  - 各种快捷配置
  - 配置复制程序处理，在副手时，主手方块设备放置时自动使用配置
  - 模组教程
- 灵能锻造合成表、灵能合成台、祭品台、强化台等 api
- 法术
  - 数值平衡
    - 灵力消耗->伤害
      - 基础法术有着顶级的 灵力->伤害 转换，并以此为平衡性基准
    - 物理伤害不能百分百抵抗，作为后期伤害来源
    - 前期魔法伤害更强，中期魔法抗性崛起，后期各种魔法百分百抗性
  - 法术通道（元素灵根）
    - 法术会需求多条或不需求（基础法术）通道，消耗同理
    - 自恢复属性，但比灵能要少短得多，用于作为法术冷却机制，有自恢复上限和最大上下限（可超载或为负）
    - 少数法术对通道的需求和消耗不一样，比如需要 100 但只消耗 10，别忘了需求必然不小于消耗
    - 可通过通道给予法术增益，法杖通常都会提供
    - 硬编码一个隐藏通道用于限制连发速度
  - 中间状态
    - 类似于末影人手拿方块的这个状态，作为法术使用的中间状态，通过滚轮切换物品栏退出
    - 抓取技能在抓取后就进入这个状态，此时攻击会投掷，长按蓄力，互动会放置
  - 灵能等级
    - 一个类似于经验等级的的灵能等级
- 随机名字 api
  - 不同步，各有表
- 产线设计器
  - 包含世界标注等使用功能
- 箱庭 api
- 调整灵能 icon

- 尽量减少起源注册的物品方块 挪到下面

# 狂野工艺

- 法术
  - 全局
    - 很多技能都可以在互动距离上（已选择目标）跳过法术弹直接应用效果
    - 法术弹本身也耗灵力和召唤灵根
  - 抓取投掷
    - 抓方块
      - 抓出来是挖掘掉落的方块，石头抓出来会变圆石，抓冰能抓但会消失，等于破坏了
      - 精准采集，抓出来的附加精准采集判定，能抓出冰
      - 超级精准，保持方块本身
      - 投掷的方块是下落方块的子类
      - 基础投掷伤害受是否镐挖、是否完整方块、硬度影响，可额外覆盖
      - 粉碎投掷，加伤，损坏方块，部分方块有额外增伤，比如玻璃
    - 抓生物
      - 技能等级越高，能抓的生物越强
      - 投掷的生物会受到撞击伤害
      - 部分生物如tnt和苦力怕在命中后会直接引爆，不管是被扔的还是被砸的
    - 蓄力投掷
    - 多物投掷，可抓取多个东西，一起投掷出去
    - 潜行时，抓住的东西会举在正前面，可阻挡弹射物等
  - 魔法护盾
  - 耐久消耗取决于抗住了多少
    - 软盾，强攻击会直接穿透，但消耗的耐久也会变少，易被饱和弱攻击击碎，除非有吸收能力
    - 硬盾，比软盾更能抗，抗下了就是百分百抗住，但强大的攻击会消耗更多耐久，易被强大的攻击击碎，不支持吸收攻击
  - 冰火系
    - 虽然有冲突，但可以上脆弱（具体怎么上未定）
    - 效率回收技能：用冰时增加热力通道，用火时增加冰力通道
  - 召唤系，召唤缠绕敌人的植物、触手，召唤打手，召唤大石头配合投掷技能等
- 灵石（货币）
- 材料
  - 黄铜，塑料，强化塑料，用作设备外壳
  - 中级导魔金属：原版金，用于设备内部
  - 高级导魔金属（紫色的金，暂叫玄金），用于魔法道具
- 产能类设备，缺名
  - 辐射灵能
    - 宇宙辐射，太阳能
  - 燃料灵能
    - 消耗燃料，不止传统燃料，不一定要烧的
    - 天然气、沼气
  - 核力灵能
    - 需要起步灵力，不多。
    - 本质上就是爆核弹，只是利用吸收类魔法将反应控制住
    - 裂变聚变均可
- 灵能锻造台
  - 消耗灵力合成物品，只能合成瞬间合成和不超过需求的储存能量的东西
- 高级灵能锻造台
  - 消耗大量灵力合成高级物品，自由多方块结构
  - 配套的祭品台放在范围内自动连接，不限摆放形状
  - 两种合成台使用相同的合成表，利用普通的只有 9 格，和能量储存来区分
    - 加速升级，给台加速度 (叫增压升级)
    - 并行效率升级，并行数*合成时间*根号 2，3，4
- 特化专用合成机器，比灵能合成台更省灵力和材料
- 分子结晶器，分子级打印，制作高精度物品。
- 魔法水晶、石碑，类似于传统机器。
  - 技术上需要考虑谁用模拟玩家，或者两者都用
  - 可加载区块
  - 附魔分离器、耕种器
- 幽灵基站：
  - 比单水晶能执行更多任务。
  - 用于召唤和自动化工具幽灵，在基座上设置幽灵在哪里用什么法术，和物品配置。幽灵挂了后扣灵力自动重生。幽灵需要的物品和灵力由基座给
  - 可以设置术式施放时间线，使幽灵按照时间线执行多种法术。
  - 术式执行延迟可以选择开始执行后，执行完成后，延迟可选时间执行下一条术式。
  - 可加载区块
- 自动化专用术式，比如移动到哪里。
- 灵魂，装了灵魂的灵魂水晶叫灵魂。id 强迫症完美方案，太棒了。一般用来合成
- 区块加载器
- 自动合成器
  - 每一个槽都可以配置过滤，满槽自动合成
- 生物
  - 幽灵
    - 可以穿墙
    - 友好幽灵和迷失（走火入魔）幽灵
    - 友好幽灵能交易，能指挥做简单事情，不愿意当黑奴
    - 能驯服，驯服后设置跟随能加载区块，防止意外丢失
    - 机器幽灵没有工作限制
  - 猫娘
    - 可以信任，不可驯服
    - 可交易
    - 会召唤幽灵
- 搞笑物品
  - 首雷蛋
    - 生物头插个引雷针塞个蛋合成
  - 脚雷（地雷）
    - 初始为方块，需要坚固上表面
    - 用下方方块能用铲子挖，用铲子右键后变为实体，陷入方块里面
- 魔法物品
  - 法师的灵力可直接供应其内部能源
  - 灵能枪
    - 蓄力枪自动蓄一格，作为射速
    - 手枪，两段蓄力
    - 冲锋枪，高射速
    - 步枪，射程伤害精准
    - 狙击，多段蓄力，每一段多穿透一格方块
    - 肩扛炮，蓄力越多爆炸越炸
    - 比法杖节省灵力，可使用灵能匣
  - 变形法杖（变形核心）
  - 千魂幡，人皇幡
    - 无损储存若干灵魂，整合大量灵魂
    - 当然，能施法
- 试炼维度，各种副本，不可挖掘
  - 奖励定制化武器装备，通常作为 boss 奖品，不可自定义，或者限制自定义
- 渡劫升级机制
  - 玩家初始无灵能不可修炼，必须进行渡劫仪式才能获得灵能成为法师进行修炼
  - 10、30、60级需要渡劫来突破等级上限，暂定最高等级100，分别为雷劈3、6、10下
- 灵魂融合机制
  - 需要已成为法师
  - 融合其他灵魂，改变自身天赋数值，可从魂幡提取自定义灵魂
- BOSS
  - 凶猫，星言，时雨，（名字暂定）
  - 吸血雌小鬼

# 野性物流

### AE系自动化痛点（不含被动产线）
- 样板写到吐
  - 其实也就高难度科技模组要写多点
- 阵法合成难
  - 子网+储存总线+总线，臃肿
  - 模块和路由器（稍微慢点），超立方（要一个个配置），而且对中心物品可以是原料也可以是成品的东西无解，只能分阵
  - 需要红石信号或者其他条件时要搞生电或者其他模组
  - 高级ae的供应器不能远程
- 合成时供应输出规则不够专一
  - 合一点输出一点对一些机器很不友好
  - 期望是将需要的各种资源统一合成，比如先合成所有锭，再合成所有块，而不是合成9个锭然后合成一个块循环。
  - 格雷的样板仓能做到，但仅支持格雷
- 供应输出阻塞规则缺点
  - 其实正在合成合成的东西不需要阻塞，仅阻塞其他合成表才是更普遍的，但需要先解决上一个问题
- 不支持的供应输出
  - 带催化剂（并非什么催化剂都可以一股脑塞机器里）
  - 概率消耗和产出（被动产线不在讨论范围）
  - 跨样板前后循环增值的
- 无复合过滤
  - 即便是联动模组都没
  - 进行复杂的分拣系统，不但要建一大堆子网，子网还相互缠绕，还需要搭配其他模组，非常混乱。
- 点对点不能多对多
  - 使用模组超立方体
- 磁盘储存特性
  - 需要好好规划储存与分类，轻量整合包还行，大型整合包就是屎机制
  - 倒是给点机制自动整理磁盘啊
  - 其实驱动器墙也不是不行，经典暴力解法了

### 假如真要另起炉灶

- 库存整合器，类似汤姆储存的终端
  - 前期工具，只能整合库存，无它功能
  - 只有一个库存整合规则，并且不支持组合规则
- 仓管石碑
  - 兼容ae，可作为ae驱动器，也可仅拉取ae网络
  - 仓管接口
    - 库存公开规则
      - 空的黑名单会公开整个库存
      - 使用白名单公开特定资源
    - 资源供应规则
      - 类似ME接口的独立库存供应，但可以配置补给间隔
    - 合成供应规则
      - 标记机器、输出输入口，以及合成表类型和过滤，仓管会自动识别合成
      - 或者自定义合成表
  - 通道，过滤规则
    - 需要一种机制使得在ui里进出资源和磁铁、自动掠夺类输入分开分拣
    - 仓管接口可以复用
- 路由水晶
  - 路由资源
- 点对点连接器
  - 用于快速建立点对点资源网络，贴上就能用，可以简单配置一下优先级输出输入限制
- 宝库
  - 让仓管创建（消耗物品）和托管，一点点灵力当维护费，本质作为虚空库存，仓管只是连接上，破坏仓管不会消失，可撤回物品（必须为空），然后拿去合成
  - 宝库升级 (升级槽数受等级影响)
    - 升级运作时耗能！
  - 原型宝库（archetype_vault），禁止 nbt
    - 水生成升级，初始两桶水，每水流动时间将当前水加倍。
    - 石生成升级，根据频道内岩浆、灵魂土、蓝冰存量，每岩浆流动时间生成等量圆石、石头、玄武岩（？）
    - 岩浆生成升级，根据岩浆、滴水石块、滴水石尖的存量，每 60 秒生成？1B*x 岩浆。
    - 种植园升级，根据种植园升级专用合成表生产农作物，能自定义并行上限以免耗光灵力。
    - 拆包解包升级
  - 装备宝库
    - 自动修理升级，自动合并储存的装备，自动经验修补
    - 自动导出附魔，联动狂野工艺里的附魔分离器。
- 仓管物品 app，提供弹药，补充物品，自动存物，99 快捷栏
- 传送石碑
  - 传送点有全局、区域、本地层级，默认展示同区域下的本地传送点
  - 需要附近有信标，才可跨区域、跨维度、远距离
  - 提供自然生成结构，多为破败版
